# LogicDraw - 新一代 Vibe Coding 平台

> **核心理念**：所画即所得，逻辑可溯源，告别模糊提示词，重掌代码控制权。

## 🌟 项目愿景

LogicDraw 旨在构建一个通过可视化逻辑图（由简易流程图 + 少量文字描述）直接生成全栈代码的智能开发平台。它不仅仅是一个代码生成器，更是一个能够将你的思维导图完美转化为生产级代码的系统。我们致力于让 AI 100% 实现你所设想的项目，同时保持底层逻辑的透明与可修改性。

## 🎯 解决痛点 (The "Why")

目前的 "Prompt Coding" (提示词编程) 或 "Vibe Coding" 存在两个致命的核心问题：

1.  **模糊性困境 (The Ambiguity Problem)**：
    自然语言提示词（Prompt）在描述复杂逻辑时往往显得苍白无力且存在歧义。AI 常常"猜"不对你想要的具体交互细节，导致生成结果总是差点意思。
    
2.  **维护性黑洞 ( The Maintenance Trap)**：
    一旦 AI 生成了项目，如果后续需要修改底层逻辑（例如改变一个状态机的流转条件），往往异常困难。你不得不重新调整提示词并祈祷 AI 不会破坏现有的功能——这使得生成的项目往往是一次性的，难以维护和迭代。

## 💡 解决方案：LogicDraw

LogicDraw 通过引入**可视化逻辑层**，彻底改变这一现状：

### 1. 后端：逻辑图驱动 (Logic-to-Backend)
*   **输入**：用户绘制简易的逻辑关系图、工作流或状态机图，辅以少量文字说明关键参数。
*   **处理**：AI 深度解析图结构，理解业务流转。
*   **输出**：自动生成结构严谨、逻辑清晰的后端代码（API、数据库模型、业务逻辑层）。
*   **优势**：业务逻辑不再隐藏在复杂的 Prompt 中，而是直观地展示在画布上。

### 2. 前端：交互流驱动 (Flow-to-Frontend)
*   **输入**：用户绘制用户交互流程图（User Interaction Flow）或界面跳转流。
*   **处理**：AI 识别页面结构与交互触发条件。
*   **输出**：生成完整的前端工程，包含 UI 组件、路由配置及与后端的对接逻辑。
*   **优势**：不仅仅是画皮（UI），更是画骨（交互逻辑）。

### 3. 白盒化与可控性 (White-box & Controllability)
*   **代码即逻辑**：生成的代码不再是不可修改的"黑盒"。我们生成的每一行代码都旨在易于阅读和修改。
*   **脱离提示词依赖**：项目的核心不再是那段几千字的 Prompt，而是清晰的业务逻辑图。
*   **底层可改**：一旦项目完成，你可以像维护传统项目一样，直接深入代码底层修改逻辑，或者更新流程图重新生成，系统都能从容应对。

## 🚀 核心工作流 (Workflow)

1.  **Draft (绘图)**：在 LogicDraw 画板上勾勒你的后端逻辑流和前端交互流。
2.  **Generate (生成)**：AI 引擎分析视觉信息，自动构建全栈项目结构。
3.  **Run (运行)**：一键启动应用，见证你的设想 100% 落地。
4.  **Evolve (演进)**：发现逻辑漏洞？直接修改图或代码，保持项目的持续生命力。
